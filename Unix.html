Sistena Operativo Unix

HSTORIA Y EVOLUCION DE UNIX

Unix fue concebido en los laboratorios Bell de AT&T por un grupo de desarrolladores liderado por Ken Thompson y Dennis Ritchie. La década de 1960 marcó los primeros esfuerzos, con la creación de herramientas como el ensamblador B y el sistema operativo Multics.

BSD y Variantes (1977 en adelante):

La Universidad de California, Berkeley, desarrolló el proyecto BSD (Berkeley Software Distribution) como una variante de Unix con mejoras significativas. BSD se convirtió en un sistema independiente y sirvió como base para sistemas operativos como FreeBSD y OpenBSD.

Guerra de Sistemas Operativos (1980-1990):

Durante la década de 1980, Unix compitió con otros sistemas operativos, como VMS (Virtual Memory System) de Digital Equipment Corporation y las minicomputadoras. La accesibilidad y la portabilidad de Unix jugaron un papel crucial en su adopción.

GNU y el Movimiento de Código Abierto (1983 en adelante):

Richard Stallman inició el Proyecto GNU para desarrollar un sistema operativo compatible con Unix pero completamente de código abierto. Aunque el kernel GNU Hurd no tuvo tanto éxito, el sistema operativo GNU se combinó con el kernel Linux para formar una solución completa.

Linux (1991 en adelante):

Linus Torvalds lanzó el kernel Linux en 1991, y su combinación con el sistema GNU creó una alternativa gratuita y de código abierto a Unix. La popularidad de Linux ha crecido enormemente, especialmente en entornos de servidor y sistemas embebidos.

Estructura del Sistema de Archivos

Organización Jerárquica:

El sistema de archivos de Unix sigue una estructura jerárquica con un directorio raíz ("/") que contiene subdirectorios como /bin, /etc, /home, entre otros. Esta organización facilita la ubicación y gestión de archivos.

Sistema de Archivos Virtuales (VFS):

VFS es una abstracción que permite que varios sistemas de archivos coexistan, independientemente de su implementación subyacente. Esto facilita la compatibilidad y la manipulación de diferentes tipos de sistemas de archivos.

Montaje y Desmontaje de Sistemas de Archivos:

Unix permite montar y desmontar sistemas de archivos en puntos específicos del sistema de archivos. El comando mount se utiliza para asociar un sistema de archivos con un directorio, mientras que umount se utiliza para desmontarlo.

Sistemas de Archivos Comunes:

Unix admite varios sistemas de archivos, cada uno con sus propias características. Algunos ejemplos incluyen:

  • ext4: Sistema de archivos extendido 4, común en sistemas Linux.
  • ZFS: Sistema de archivos de Zettabyte File System, conocido por su integridad de datos y capacidades avanzadas de almacenamiento.

Interfaz de Usuario y Shell

Introducción a la Interfaz de Línea de Comandos:

La interfaz de línea de comandos (CLI) es una parte integral de Unix. Los usuarios interactúan con el sistema ingresando comandos en un shell, como Bash o Zsh. La CLI proporciona flexibilidad y automatización a través de scripts.

Funciones y Tipos de Shells:

Los shells son intérpretes de comandos que procesan las entradas del usuario y ejecutan programas. Bash (Bourne Again SHell) es uno de los más comunes, pero existen otros como Zsh, Ksh y Fish, cada uno con sus propias características.

Personalización de la Línea de Comandos y Scripts:

Los usuarios pueden personalizar su entorno de línea de comandos mediante la configuración de archivos como .bashrc o .zshrc. Además, la creación de scripts permite automatizar tareas y mejorar la eficiencia del trabajo diario.

Gestión de Procesos

Conceptos de Procesos en Unix:

Un proceso en Unix es una instancia en ejecución de un programa. Cada proceso tiene un identificador único (PID), y el sistema operativo gestiona su ejecución y recursos asociados.

Comandos para Administrar Procesos:

  • ps: Muestra información sobre los procesos en ejecución.
  • kill: Envía señales a un proceso para controlar su ejecución.
  • top y htop: Muestran información en tiempo real sobre el uso del sistema y los procesos.

Programación de Tareas con cron y at:

  • cron: Permite la ejecución programada de tareas en intervalos regulares.
  • at: Programa la ejecución de tareas en un momento específico.

Seguridad y Permisos

Modelo de Permisos de Unix:

Los permisos en Unix controlan el acceso a archivos y directorios. Cada archivo tiene propietario, grupo y permisos para el propietario, el grupo y otros usuarios.

Gestión de Usuarios y Grupos:

  • useradd y userdel: Añaden y eliminan usuarios.
  • groupadd y groupdel: Añaden y eliminan grupos.
  • passwd: Cambia la contraseña de un usuario.

Acceso Remoto y Seguridad en la Red:

  • ssh: Protocolo seguro para acceder a sistemas remotos.
  • iptables y firewalld: Herramientas para configurar reglas de firewall.
